# 安全与容灾

## 1. 安全设计

### 1.1 网络隔离

| 区域 | 访问策略 |
|------|----------|
| 摄像头网 → ZLM RTP | 仅允许摄像头网段 → 对应端口段 |
| 公网 → Media Edge | 仅 443（+ 可选 1935 专线） |
| 公网 → SRS/ZLM 直连 | **禁止** |
| Business → MEDIA API | 内网 VLAN40 |
| Upload Worker → MinIO | 内网 VLAN30，专用 AK/SK |
| 运维 → SRS API 1985 | VPN + IP 白名单 |

### 1.2 鉴权

| 链路 | 措施 |
|------|------|
| RTMP 推流 | MEDIA `on_publish` 校验 device_id；可选 token |
| HTTP-FLV 播放 | Edge Nginx auth_request 调 iot-gateway；或 signed URL |
| MinIO | 独立 service account，最小权限 bucket |
| MEDIA API | 内网 mTLS 或 API Key（Header `X-Media-Token`） |
| Kafka | SASL/SCRAM |
| GlusterFS | 仅媒体 VLAN 可 mount |

### 1.3 密钥管理

- ZLM `secret`、SRS Hook、MinIO AK/SK 放 **Vault** 或 K8s Secret
- 禁止写入 Git；模板用 `${ZLM_SECRET}` 占位

### 1.4 审计

- MEDIA：`allocate` / `migrate` / `admin/*` 写审计日志
- MinIO：开启 audit webhook
- 录像访问：沿用现网 gateway 鉴权

## 2. 高可用

| 组件 | HA 方案 | RTO | RPO |
|------|---------|-----|-----|
| MEDIA Scheduler | 3+ 无状态 + LB | < 1min | 0 |
| SRS | 多节点 + 设备级 migrate | 5min | 流中断可接受 |
| ZLM | 多节点 + WVP 重选 | 3min | 同上 |
| GlusterFS | Replica 3 | 分钟级 | 缓冲数据可重建 |
| Kafka | 3 broker, RF=3 | < 5min | 0（未消费消息保留） |
| MinIO | EC 4+4 | 节点级 | 0 |
| Upload Worker | 10+ 实例 | 秒级 | at-least-once |

## 3. 容灾场景

### 3.1 单机房（本方案默认）

- 所有组件同机房不同机架
- MinIO EC 跨机架

### 3.2 双机房（扩展）

| 组件 | 策略 |
|------|------|
| MinIO | site replication 到备机房 |
| 数据库 | 流复制 |
| 媒体 | 备机房冷备 SRS 池，DNS  failover |
| GlusterFS | **不跨机房**；备机房独立缓冲卷 |

### 3.3 灾难恢复

1. MinIO 从备份/restic 恢复对象
2. PostgreSQL PITR
3. 媒体层：新部署 SRS/ZLM 池 + 全量 `migrate_stream_bindings.py`
4. 设备侧重连推流（摄像头自动重推）

## 4. 数据保护

| 数据 | 位置 | 保护 |
|------|------|------|
| 录像（长期） | MinIO | EC + 备份 |
| 录像（缓冲） | GlusterFS | 可丢失，可从流重录 |
| 抓拍 | MinIO | 同录像 |
| 流绑定 | PostgreSQL | 备份 |
| 节点状态 | Redis | 可重建 |

## 5. 合规建议

- 录像 retention 遵循 `RecordSpace.save_time`
- 删除设备时：解绑 MEDIA + 清理 MinIO prefix + 停推流
- 日志不含敏感视频内容

## 6. 威胁模型摘要

| 威胁 | 缓解 |
|------|------|
| 未授权推流占用带宽 | on_publish 鉴权 |
| 未授权播放 | Edge auth / signed URL |
| MinIO bucket 公开误配置 | 沿用 `ensure_bucket_public_read_write_policy` 审计 |
| Kafka 消息注入 | 内网 + SASL；Worker 校验 file_path 在允许目录内 |
| GlusterFS 路径遍历 | Worker 校验路径 prefix |
