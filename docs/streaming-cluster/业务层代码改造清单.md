## 2. VIDEO 模块

### 2.1 P0 — 必须改造

#### `app/services/camera_service.py`

**现状**：`_default_stream_urls()` / `_get_host_ip_for_stream_urls()` 硬编码宿主机 IP 与端口。

**改造**：

```python
def _default_stream_urls(device_id: str) -> tuple[str, str, str, str]:
    if current_app.config.get('MEDIA_SCHEDULER_ENABLED'):
        from app.services.media_client import get_media_client
        data = get_media_client().allocate_or_get_urls(device_id)
        u = data['urls']
        return u['rtmp_stream'], u['http_stream'], u['ai_rtmp_stream'], u['ai_http_stream']
    # 仅开发环境保留 legacy
    host = _get_host_ip_for_stream_urls()
    ...
```

**触发点**：

- `register_camera()` / 创建设备
- `update_camera()` source 变更时重新 `allocate`
- 批量导入脚本

#### 新建 `app/services/media_client.py`

封装 MEDIA API（见 02 文档），带本地缓存（TTL 60s）与熔断。

#### `app/blueprints/camera.py`

| 路由 | 改造 |
|------|------|
| `on_dvr_callback` | `MEDIA_UPLOAD_MODE=kafka` 时仅记录日志返回 200；或完全移除 Hook 注册 |
| `on_publish_callback` | 逻辑迁移至 MEDIA `/hook/srs/on_publish`；VIDEO 保留兼容转发 3 个月 |
| `_resolve_playback_absolute_path` | `/data` → `MEDIA_HOST_DATA_ROOT` |

#### 新建 `services/media_upload_worker/`

从 `on_dvr_callback` 抽取：

- MinIO 上传
- `Playback` 创建
- 封面 ffmpeg
- 告警 `record_path` 回写

依赖：Kafka consumer、Flask app context 或独立 DB session。

#### `.env` / `.env.prod` / `.env.docker`

新增：

```bash
MEDIA_SCHEDULER_URL=http://10.10.40.10:8090
MEDIA_SCHEDULER_ENABLED=true
MEDIA_RECORD_DIR=/mnt/easyaiot-media/playbacks
MEDIA_HOST_DATA_ROOT=/mnt/easyaiot-media
MEDIA_UPLOAD_MODE=kafka
KAFKA_BROKERS=10.10.40.21:9092
```

## 7. 数据迁移脚本

新建 `VIDEO/scripts/migrate_stream_bindings.py`：

```python
"""
批量为已有设备调用 MEDIA /bindings/allocate，刷新 rtmp_stream 等字段。
用法: python scripts/migrate_stream_bindings.py --batch-size 500 --dry-run
"""
```

步骤：

1. `SELECT id FROM device ORDER BY id`
2. 分批 POST `/bindings/allocate`
3. UPDATE device SET rtmp_stream=..., http_stream=..., ...
4. 输出失败列表

**注意**：迁移时需 **停推流窗口** 或逐批迁移并重启算法任务。
