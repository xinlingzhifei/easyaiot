## 2. 操作系统基线

- Ubuntu 22.04 LTS / Rocky Linux 9
- 内核参数：

```bash
sysctl -w net.core.rmem_max=16777216
sysctl -w net.core.wmem_max=16777216
sysctl -w net.ipv4.ip_local_port_range="1024 65535"
sysctl -w fs.file-max=2097152
```

- 关闭 firewalld 或精确放行（见 01 文档端口表）
- 时间同步：chrony，全集群 < 50ms 偏差

## 3. 部署顺序

```
1. VLAN / 交换机 / DNS
2. GlusterFS 集群 + 卷创建 + 客户端挂载验证
3. MinIO 分布式集群
4. Kafka 集群 + topic 创建
5. PostgreSQL / Redis（若未有）
6. MEDIA Scheduler ×3
7. Upload Worker ×N（先 2 实例验证）
8. SRS 节点池（先 3 节点 POC）
9. ZLM 节点池
10. Media Edge Nginx
11. Business 集群（VIDEO/DEVICE）+ 配置 MEDIA_SCHEDULER_URL
12. 迁移脚本 + 灰度
```

## 4. 单 SRS 节点部署步骤

```bash
# 1. 挂载 GlusterFS
bash .scripts/media-cluster/glusterfs/mount-all.sh

# 2. 渲染配置
export MEDIA_NODE_ID=srs-live-01
export MEDIA_SCHEDULER_HOST=10.10.40.10
export SRS_CANDIDATE_IP=10.10.20.11
envsubst < .scripts/media-cluster/srs/cluster.conf.template > /opt/srs/conf/docker.conf

# 3. 启动
docker compose -f .scripts/media-cluster/docker-compose.media-node.yml up -d

# 4. 注册
bash .scripts/media-cluster/srs/register.sh

# 5. 验证
curl http://10.10.20.11:1985/api/v1/versions
```

## 7. 日常运维

### 7.1 扩容 SRS 节点

1. 新机器挂载 GlusterFS
2. 部署 SRS，`MEDIA_NODE_ID=srs-live-17`
3. `register.sh` 自动加入 ZSet
4. **新设备**自动分配；旧设备不迁移除非 rebalance

### 7.2 缩容

1. `POST /admin/drain/{node_id}` 停止新绑定
2. 等待 active_streams → 0
3. 迁移绑定 `POST /bindings/migrate`
4. 下线容器

### 7.3 GlusterFS 维护

```bash
gluster volume status gv-playbacks
gluster volume heal gv-playbacks info
df -h /mnt/easyaiot-media/playbacks
find /mnt/easyaiot-media/playbacks -name '*.flv' | wc -l   # 应维持低位
```

### 7.4 Kafka lag 处理

```bash
kafka-consumer-groups.sh --bootstrap-server ... --describe --group upload-worker-dvr
```

lag > 10000：扩容 Worker；检查 MinIO 是否瓶颈。

## 8. 故障处理 Runbook

### 8.1 播放黑屏

1. 查设备 `http_stream` 指向哪个 SRS
2. `curl http://{node}:8080/live/{id}.flv` 是否 200
3. SRS API 查流是否存在
4. 算法是否在推 live/ai
5. Edge nginx error log

### 8.2 DVR 不上传 MinIO

1. Kafka topic 是否有新消息
2. Worker 日志 / DLQ
3. GlusterFS 文件是否堆积
4. MinIO 健康检查
5. hybrid 模式下对比 legacy on_dvr

### 8.3 GlusterFS 满

1. 紧急：Janitor 按 mtime 删未上传文件（极端）
2. 正常：扩容 Worker + 检查 MinIO 写入速率
3. 增大 `dvr_duration` 降低文件数

### 8.4 SRS 节点宕机

1. MEDIA 标记 offline
2. 影响设备列表：`SELECT * FROM media.device_media_binding WHERE srs_live_node_id='...'`
3. 批量 migrate API
4. 重启算法任务 / 推流转发

### 8.5 国标点播失败

1. ZLM 节点 RTP 端口是否耗尽
2. WVP 选中节点是否 online
3. `stream-ip` / `sdp-ip` 是否正确（公网/专网）

## 12. 回滚方案

| 阶段 | 回滚 |
|------|------|
| Phase 1 存储 | DVR 路径改回本地 `/data`，停 Worker |
| Phase 2 Hook | SRS Hook 指回 VIDEO on_dvr |
| Phase 3 集群 | DNS 指回单机 SRS IP；DB URL 批量回滚脚本 |

保留 legacy 路径 **至少一个版本周期**。
